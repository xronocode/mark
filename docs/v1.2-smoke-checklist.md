# v1.2.0 Functional Smoke Checklist

Pre-release verification for the renderer hardening track. Every flow
below was working in v1.1.0 and **must still work** in v1.2.0 after the
contextIsolation / nodeIntegration flip.

Run through this on a clean dev build (`npm run dev`) or a packaged DMG.
Mark each item PASS / FAIL, note any visible regression. If anything
fails, `git revert f8539737` (the security flip) buys time to investigate
without blocking other work.

---

## Critical (validates the audit findings)

- [ ] **C-1** — Preferences → Image → uploader = `cliScript` → enter a path
  to an existing executable file (e.g. `/usr/bin/true`) → verify the
  Save / OK button enables. Then enter a path to a non-executable file
  (e.g. `/etc/hosts`) → verify the button disables. *(Validates audit
  blocker B-1: stat.isFile() across contextBridge.)*

- [ ] **C-2** — Paste an image from the clipboard while uploader =
  `picgo`. Image should upload via main-process exec (no renderer
  child_process). Verify the resulting Markdown image link points at
  the uploaded URL. Check that
  `~/Library/Caches/com.xronocode.mark/` does **not** accumulate
  tmpfiles after several pastes. *(Validates the unlink fix in
  `99cb11d2` and the picgo relocation in 8h.)*

- [ ] **C-3** — Open a folder, run a content search across at least 50
  files. Results must stream into the panel as ripgrep finds them, not
  all at once at the end. Cancel mid-search via the close-search
  button. *(Validates the streaming IPC search in 8i.)*

- [ ] **C-4** — Right-click a tab → context menu shows
  Close / Close Others / Close Saved / Close All / (separator) /
  Rename / Copy Path / Show in Folder. Each item invokes the correct
  action. *(Validates the serialized context-menu IPC in 8g.)*

- [ ] **C-5** — Right-click in the sidebar (file or folder) → context
  menu shows New File / New Directory / Copy / Cut / Paste / Rename /
  Move to Trash / Show in Folder. PASTE must be enabled only when the
  hasPathCache (Cmd+C / Cmd+X) buffer is non-empty. *(Validates the
  hasPathCache enabled flag survives serialization.)*

## How to record results

Save a copy of this file as `docs/v1.2-smoke-results-<YYYY-MM-DD>.md`,
fill in PASS / FAIL boxes inline, and add a `Notes` section per
failed item with reproduction steps + a screenshot if visible.

Recommended order: Critical (C-1..C-5) first → if any fail, stop and
revert. Then the rest in any order.

## Failure runbook

If anything in **Critical** fails:

1. Capture the renderer console error (Cmd+Option+I → Console tab).
2. Note the exact step that triggered it.
3. `git revert 99cb11d2 f8539737` to roll back the security flip
   (the renderer code already worked under contextIsolation:false in
   step-8a..8m).
4. Re-run the failing item to confirm it passes pre-flip.
5. Open an issue / report so the underlying preload bridge can be
   patched without unwinding the rest of the hardening.

If a non-Critical item fails:

1. Run with `MT_DEBUG=1 npm run dev` to get verbose preload logging.
2. The audit doc lists each sub-ticket's commit hash; targeted
   `git revert <hash>` is usually enough to bisect.

## Out of scope for v1.2.0 (will be v1.3 hardening track)

- `webSecurity: true` (custom protocol handler for file:// images +
  themes).
- Removing the `contextIsolation:false` fallback branches in preload.
- Replacing `arrayBufferToBase64` with chunked TextDecoder.
- Vendoring `@hfelix/electron-localshortcut` utility files (currently
  works via Vite `define` for `process.platform`).
